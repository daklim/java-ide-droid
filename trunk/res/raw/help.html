JavaIDEdroid

1. Introduction

JavaIDEdroid is an Android application that aims to be an integrated development environment for creating native Android applications. Ideally, it will allow you to create APK files completely on Android devices, without the need to use the Android SDK on Windows or Linux.

Version:0.7.0
Date: 2011-06-02
Author:Tom Arn, www.t-arn.com
Contributions by: Ken Ellinwood
Project home page:http://code.google.com/p/java-ide-droid/

Currently, following tools are integrated into JavaIDEdroid:

  • BeanShell (www.beanshell.org)
  • Eclipse Java Compiler (www.eclipse.org)
  • Dx tool from the Android SDK (developer.android.com)
  • ApkBuilder tool from the Android SDK (developer.android.com)
  • Zipsigner-lib from Ken Ellinwood (http://code.google.com/p/zip-signer)

2. BeanShell

The integrated BeanShell interpreter allows you to automate the build process by writing your own BeanShell scripts.

2.1 Writing scripts

BeanShell scripts can access the functionality built into JavaIDEdroid by using the pre-defined variable mainActivity. This variable references the MainActivity class with following accessible methods:

  • int ide.fnCompile (String arguments)
  • int ide.fnCompile (String[] arguments)
  • int ide.fnDx (String arguments)
  • int ide.fnDx (String[] arguments)
  • int ide.fnApkBuilder (String arguments)
  • int ide.fnApkBuilder (String[] arguments)
  • int ide.fnSignApk (String arguments)
  • int ide.fnSignApk (String[] arguments)
  • String[] ide.fnTokenize (String commandline) // tokenizes the commandline arguments
  • void bshTask.fnClear () // clears BeanShell output
  • void bshTask.fnPublishProgress (String msg) // updates progress dialog message
  • void bshTask.fnToast (String msg, int milliseconds) // shows a toast message

Meaning of the return value: 

   0:   OK
   >0:  warning or error
   255: no defined return value

Example for calling ecj:
int rc = mainActivity.ide.fnCompile("--version");

2.2 Running scripts

BeanShell scripts must be stored somewhere on the Storage Card. You can enter the path and filename of the script in the edit field or choose the script by tapping the folder icon. To choose a script, the OI File Manager needs to be installed on the device. The "Run script!" button will start the specified BeanShell script.
