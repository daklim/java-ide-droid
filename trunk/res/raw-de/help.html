JavaIDEdroid

1. Einleitung

JavaIDEdroid ist eine integrierte Softwareentwicklungsumgebung (IDE), welche auf Android Geräten läuft und es ermöglicht, native Android Anwendungen zu erstellen, ohne den Android SDK auf Windows oder Linux verwenden zu müssen. Der IDE sollte auf Android 1.6 oder höher lauffähig sein.

Version:1.1.0
Datum: 2011-08-31
Autor:Tom Arn, www.t-arn.com
Beiträge von: Ken Ellinwood
Projekt Webseite:http://code.google.com/p/java-ide-droid/

Zurzeit sind folgende Tools in JavaIDEdroid integriert:

  • Aapt Tool aus dem Android SDK (developer.android.com)
  • Eclipse Java Compiler (www.eclipse.org)
  • Dx Tool aus dem Android SDK (developer.android.com)
  • ApkBuilder Tool aus dem Android SDK (developer.android.com)
  • Zipsigner-lib von Ken Ellinwood (http://code.google.com/p/zip-signer)
  • BeanShell Interpreter (www.beanshell.org)

Aapt ist integriert als native Shared Library (arm). Es verwendet externe Libraries, welche Android 2.3 oder höher benötigen, weshalb die aapt Shared Library für Android 2.3 erstellt wurde. Trotzdem scheint aapt auch auf älteren Android Versionen zu laufen (es lief problemlos auf meinem Android 2.2.1).

2. BeanShell

Mit dem integrierten BeanShell Interpreter können BeanShell Skripts ausgeführt werden, um den Prozess der Anwendungserstellung zu automatisieren.

2.1 Skripts erstellen

BeanShell Skripts haben über die vordefinierte Variable G Zugriff auf die in JavaIDEdroid eingebaute Funktionalität. Die vordefinierte Variable G referenziert G.class, welche über folgende Variablen und Methoden verfügt:

  • MainActivity Klasse: G.main
  • Temporäre Passwörter: G.stPw1, G.stPw2
  • IDE Klasse: G.ide erlaubt den Zugriff auf die Entwicklungstools
  • int G.ide.fnAapt (String arguments)
  • int G.ide.fnAapt (String[] arguments)
  • int G.ide.fnCompile (String arguments)
  • int G.ide.fnCompile (String[] arguments)
  • int G.ide.fnDx (String arguments)
  • int G.ide.fnDx (String[] arguments)
  • int G.ide.fnApkBuilder (String arguments)
  • int G.ide.fnApkBuilder (String[] arguments)
  • int G.ide.fnSignApk (String arguments)
  • int G.ide.fnSignApk (String[] arguments)
  • String[] G.ide.fnTokenize (String commandline) // zerlegt die Befehlszeilen Argumente
  • BeanShellTask Klasse: G.bshTask erlaubt den Zugriff auf die Methoden für die Bildschirmausgabe
  • void G.bshTask.fnClear () // löscht die BeanShell Ausgabe
  • void G.bshTask.fnPublishProgress (String msg) // aktualisiert den Dialog mit der Fortschrittsanzeige
  • void G.bshTask.fnToast (String msg, int milliseconds) // zeigt eine Toast Meldung an
  • Setting Klasse: G.oSet erlaubt den Zugriff auf Variablen, die in BeanShell Skripts verwendet werden können:
  • G.oSet.stAndroidJarPath
  • G.oSet.stBshVar1
  • G.oSet.stBshVar2
  • G.oSet.stBshVar3
  • G.oSet.stBshVar4
  • G.oSet.stBshVar5

Bedeutung des Rückgabewerts: 

   0:   OK
   1:   Warnung
   >1:  Fehler
   99:  undefinierter Rückgabewert / 	Exception

Beispiel um ecj zu starten:
int rc = G.ide.fnCompile("--version");

2.2 Skripts ausführen

BeanShell Skripts müssen irgendwo auf der Speicherkarte gespeichert werden. Der Pfad zum Skript kann im Eingabefeld eingegeben oder über die Schaltfläche ausgewählt werden. Damit die Schaltfläche funktioniert, muss der OI File Manager installiert sein. Die Schaltfläche "Starte Skript!" startet das angegebene BeanShell Skript.

3. Tools

Dieser Reiter gibt einen direkten Zugriff auf die eingebauten Tools, ohne dass ein BeanShell Skript benötigt wird. Alle Argumente müssen im Eingabefeld eingegeben werden.

4. Verwendung aus anderen Anwendungen

JavaIDEdroid unterstützt die Intent Aktion "android.intent.action.SEND" und kann aus anderen Anwendungen mittels folgendem Code gestartet werden:

ComponentName cn = new ComponentName("com.t_arn.JavaIDEdroid", "com.t_arn.JavaIDEdroid.MainActivity");
Intent intent = new Intent();
intent.setComponent(cn);
intent.putExtra("android.intent.extra.ScriptPath", "/sdcard/build.bsh");
startActivity(intent);

Der Pfad des BeanShell Skripts kann im Extra Bundle übergeben werden. Wenn JavaIDEdroid startet, führt es dieses Skript aus.
